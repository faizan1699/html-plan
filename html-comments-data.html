HTML Comments

Comments are notes in your code that browsers ignore. They're useful for documentation and temporarily hiding code.

Basic Comment

<!-- This is a comment -->
<p>This text is visible</p>

Multi-line Comment

<!--
  This is a multi-line comment.
  You can write multiple lines here.
  Browsers will ignore all of this.
-->

Commenting Out Code

<!-- Temporarily hide this section
<div>
  <p>This content is hidden</p>
</div>
-->

Inline Comment

<p>Visible text</p> <!-- This comment is on the same line -->

Comments for Documentation

<!--
  Navigation Section
  Contains links to main pages
-->
<nav>
  <a href="index.html">Home</a>
  <a href="about.html">About</a>
</nav>

Comments for TODO Notes

<!-- TODO: Add more links here -->
<!-- FIXME: This needs to be updated -->
<!-- NOTE: Remember to check this later -->

Data Attributes (data-*)

Data attributes let you store custom data in HTML elements. They're useful for JavaScript and CSS.

Basic Data Attribute

User Info
<div data-user-id="12345" data-role="admin">User Info</div>

Accessing Data Attributes in JavaScript

<div id="myDiv" data-name="John" data-age="30">Person</div>

<script>
  var div = document.getElementById('myDiv');
  var name = div.dataset.name; // "John"
  var age = div.dataset.age;   // "30"
</script>

Data Attributes for Styling

Active Item
<div data-status="active">Active Item</div>

<style>
  [data-status="active"] {
    color: green;
  }
</style>

Multiple Data Attributes

HTML Tutorial 
<article
  data-category="tutorial"
  data-difficulty="beginner"
  data-language="html"
>
  HTML Tutorial
</article>

Data Attributes with Numbers

Product
<div data-price="29.99" data-quantity="5">Product</div>

Data Attributes for Forms

<form data-form-type="contact" data-submit-url="/api/contact">
  <input type="text" name="name" />
  <button type="submit">Submit</button>
</form>

Complete Example

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <title>Comments & Data Example</title>
</head>
<body>
  <!-- Main Navigation -->
  <nav data-nav-type="main" data-sticky="true">
    <a href="index.html">Home</a>
    <a href="about.html">About</a>
  </nav>

  <!-- Product List -->
  <section data-section="products">
    <article
      data-product-id="101"
      data-price="29.99"
      data-category="electronics"
    >
      <h2>Product Name</h2>
      <p>Product description</p>
    </article>
  </section>

  <script>
    // Access data attributes
    var nav = document.querySelector('nav');
    var navType = nav.dataset.navType; // "main"
    var isSticky = nav.dataset.sticky; // "true"

    var product = document.querySelector('article');
    var productId = product.dataset.productId; // "101"
    var price = product.dataset.price; // "29.99"
  </script>
</body>
</html>

Best Practices

  • Use comments to explain complex code
  • Keep comments up-to-date
  • Don't over-comment obvious code
  • Data attributes should start with "data-"
  • Use lowercase with hyphens for data attribute names
  • Data attributes are great for JavaScript selectors
